# 門市訂貨承諾與店務管理系統 前端實作導覽與部署說明

本系統為專屬門市的**訂貨承諾與店務管理系統**前端行動端應用，現已開發完成並通過編譯。

👉 **本說明書位置**：位於專案根目錄下的 `README.md` 檔案中。

## 🔒 帳號與登入權限管理

系統已建置完整的**「帳號與登入管理」**模組，數據持久化保存在瀏覽器的 `localStorage` 中：
*   **預設帳號與密碼**：
    *   **文和**（超級管理員）：角色 `SUPER_ADMIN`，可存取所有功能、新增/編輯所有訂單與任務。**預設密碼為空白**（直接點擊登入）。
    *   **何易俞**（分店店長）：角色 `STORE_MANAGER`，負責「台南六甲店」店務。**預設密碼為空白**。
    *   **揭怡庭**（一般店員）：角色 `STAFF`，負責櫃台店務。**預設密碼為空白**。
*   **登入管理**：支援選擇預設人員一鍵快速登入（供開發測試使用），或在輸入框中輸入個人密碼進行登入。
*   **設定中的帳號管理**：
    *   若以 **超級管理員 (文和)** 身份登入，進入「設定」頁面後，將會看到「帳號管理」專區，可在此進行 **新增帳號**、**編輯帳號名稱與所屬門市**、**修改帳密** 或 **刪除帳號** 等操作。
    *   若以 **店長或店員** 身份登入，則僅能看見「修改個人密碼」的安全區塊，無法管理他人帳號。

## 🤖 manie 智慧助手與圖示個別運用

原系統的「皮卡皮卡」趣味功能已更名為 **「manie 智慧助手」**。我們將您提供的大圖（太空裝藍髮吉祥物）透過 CSS Sprite 技術完美切分出 5 個姿勢，並巧妙地個別運用於系統各個情境中：

1.  **飛行指前方 (`welcome`)**：運用於 **登入頁面頂部** 作為歡迎指引，以及 **首頁叫醒後的 manie** 狀態。
2.  **雙手叉背後 (`back`)**：運用於 **設定頁面頂部**，代表思考後台架構。
3.  **手拿大金幣 (`gold`)**：運用於 **首頁歡迎區塊右側**，象徵門市業績與訂單承諾。
4.  **低頭鞠躬/睡覺 (`sleep`)**：運用於 **首頁尚未叫醒的 manie 互動卡片**（點擊 23 次可將其喚醒！）。
5.  **手拿平板電腦 (`tablet`)**：運用於 **任務管理頁面頂部**，象徵店員使用平板檢視並操作櫃台日常任務。

## 🛠️ 如何在本地執行與測試

請在您的終端機（如 Powershell 或 CMD）中執行以下步驟：

1.  **進入專案目錄**：
    ```bash
    cd "C:\Users\serap\Downloads\門市管理任務"
    ```
2.  **啟動本地開發伺服器**：
    ```bash
    npm run dev
    ```
    *   系統會自動開啟瀏覽器並載入網頁 `http://localhost:3000`。
    *   建議使用 Chrome 或 Edge 瀏覽器的**開發者工具 (F12)** 切換為**手機版 (RWD) 模式**，可獲得最完美的行動端 19.5:9 螢幕顯示比例！

## 📂 串接 Google 試算表部署步驟

為了與您的 Google 試算表即時連動，請按照以下步驟完成部署：

1.  開啟您的 Google 試算表，新增兩個工作表分頁並命名為 **「Orders」** 與 **「Tasks」**。
2.  將第一列的欄位標題分別設定為（可直接複製）：
    *   **Orders**: `編號`, `客戶姓名`, `客戶電話`, `商品與承諾內容`, `類型`, `分店`, `提單人員`, `客戶來源`, `客戶標籤`, `數量`, `商品單價`, `商品成本`, `到貨狀態`, `建單日期`, `預計交貨日`, `逾期天數`, `客戶簽名`, `備註`
    *   **Tasks**: `任務編號`, `時段`, `櫃台`, `任務內容`, `分數`, `是否完成`, `完成時間`, `完成人員`
3.  點選上方選單 **擴充功能 (Extensions) > Apps Script**。
4.  將專案中的 `google-apps-script.js` 指令碼內容完全複製並貼上，然後點選存檔。
5.  點擊右上角 **部署 (Deploy) > 新增部署 (New deployment)**：
    *   選取類型為「網頁應用程式 (Web app)」。
    *   將存取權限設為 **「任何人 (Anyone)」**。
    *   點擊「部署」並授權存取，複製最後產生的 **「網頁應用程式 URL」**。
6.  開啟您的前端系統網頁，點選右上角 **「設定」**：
    *   將該 URL 貼入 API 網址輸入框並點擊 **「儲存 API 設定」**。
    *   接著點擊 **「一鍵同步至 Google」**，這會將網頁目前的初始訂單與任務直接同步至您的試算表中，完成首次初始化！
